# SVG 幻灯片设计指南

读取时机：生成每张幻灯片 SVG 之前，先完成"内容分析 → 结构选择 → 视觉设计"三步思考，再动笔写 SVG 代码。

**核心原则：先理解内容的本质结构，再选择最能表达它的视觉形式。不要套模板，要让视觉为内容服务。**

## 第一步：分析内容，选择视觉结构

拿到这页的核心内容后，先问自己：**这些信息之间是什么关系？**

| 内容关系 | 视觉结构 | 典型场景 |
|----------|----------|----------|
| 多个地位相等的概念 | **并列** — 卡片、列、图标行 | 产品特性、团队成员、解决方案选项 |
| 一个核心，多个支撑点 | **总分** — 中心辐射、伞形、分支树 | 战略框架、核心论点展开、分类拆解 |
| 有顺序的步骤或阶段 | **流程** — 横向箭头、时间轴、漏斗 | 执行路径、项目阶段、转化漏斗 |
| 持续循环的过程 | **循环** — 环形箭头、齿轮、闭环 | PDCA、生命周期、反馈机制 |
| 有层级的重要性或数量 | **金字塔** — 三角形分层、阶梯 | 优先级、马斯洛需求、市场分层 |
| 两个维度的对比 | **对比** — 左右分割、天平、矩阵 | 优劣比较、现状vs目标、竞品分析 |
| 关键数字或结论 | **聚焦** — 大数字、突出色块、仪表盘 | KPI、核心结论、一句话概括 |

**选定结构后，思考如何用视觉强化这个结构**：
- 并列 → 大小/颜色相同，间距均等
- 总分 → 中心元素更大更显眼，连线指向周边
- 流程 → 箭头方向一致，步骤编号清晰
- 循环 → 弧形箭头首尾相连，暗示无限持续
- 金字塔 → 底部最宽、顶部最窄，面积代表数量/比重
- 对比 → 两侧对称，差异用颜色/大小突出

## 第二步：视觉结构图谱

### 并列结构

适用于 2–5 个平等概念。

**设计重点**：相同的卡片/容器尺寸、均等间距、同一色调但允许用强调色突出主推项。内容少（2–3 项）时卡片可以做得更大、内容更丰富；内容多（4–5 项）时保持简洁。

构建要素：背景卡片（`<rect rx>`）+ 顶部色条或图标 + 标题 + 说明文字。可以加数字序号或图标形状增加辨识度。

```
[  卡片1  ] [  卡片2  ] [  卡片3  ]
 顶部色条    顶部色条    顶部色条（强调色）
 图标/数字   图标/数字   图标/数字
 标题        标题        标题★推荐
 说明        说明        说明
```

### 总分结构

适用于一个核心概念展开到 3–6 个分支。

**设计重点**：中心元素用最大、最显眼的样式；连线/箭头从中心指向外围；外围元素大小一致。可以是"中心 + 辐射"（适合无序分支）或"顶部 + 向下展开"（适合有层级的分支）。

构建要素：中心圆形/菱形 + 连线（`<line>` 或 `<path>`）+ 周边矩形/椭圆 + 文字。

```
            [  核心概念  ]
           /      |      \
    [分支A]   [分支B]   [分支C]
```

### 流程结构

适用于 3–6 个有序步骤。

**设计重点**：方向感（通常从左到右）、步骤编号清晰可见、箭头连接各步骤。可以加时间轴、进度条。漏斗形变体（每步宽度缩窄）适合表达"转化"语义。

构建要素：步骤圆圈/矩形 + `<polygon>` 箭头 + 步骤号 + 标签 + 说明。当前步骤用强调色，其余用浅色。

```
[①步骤] →→ [②步骤] →→ [③步骤] →→ [④步骤]
  说明         说明         说明（高亮）   说明
```

漏斗变体（从上到下，逐层缩窄）：
```
  ██████████████  第一层（最宽）
    ████████████  第二层
      ██████████  第三层
         ██████   第四层（最窄）
```

### 循环结构

适用于持续往复的过程，强调"没有终点"。

**设计重点**：弧形箭头形成闭环，各节点等距分布在圆周上（3–4 个节点最佳），顺时针方向为默认。可在圆心放核心概念。

构建要素：`<circle>` 或 `<ellipse>` 底圆 + 4 个节点圆 + 弧形 `<path>` 箭头 + 圆心文字 + 节点标签。用不同深浅的同一色系表达循环感。

```
        [计划]
    ↗          ↘
[行动]  ←←←  [评估]
    ↖          ↙
        [改进]
```

### 金字塔结构

适用于表达层级、优先级或数量关系（顶部最重要/最少，底部最基础/最多）。

**设计重点**：用梯形或三角形的 `<polygon>` 分层，底部最宽、顶部最窄，每层用不同深浅的颜色（通常顶部最深）。可以是正金字塔（优先级从上到下降低）或倒金字塔（漏斗）。

构建要素：多个梯形 `<polygon>` + 层内文字 + 可选的左侧标注线。

```
     ▲  顶层（最重要）
    ▲▲  中层
   ▲▲▲  底层（基础）
```

### 对比结构

适用于两个维度的比较：A vs B、现状 vs 目标、优点 vs 缺点。

**设计重点**：左右对称布局，用颜色区分两侧（一侧冷色，一侧暖色；或一侧深一侧浅）。中间可加一条分割线或标注差异的注释。

构建要素：两个等宽矩形区块 + 各自的标题色条 + 列表内容 + 中间分割线（可选 VS 标签）。

```
[  区块 A  ] | [  区块 B  ]
  标题色条   |   标题色条
  • 特点1   |   • 特点1
  • 特点2   |   • 特点2
```

## 第三步：设计质量原则

### 视觉层级

用**大小、粗细、颜色深浅**创造清晰的视觉优先级，让观众一眼知道看什么最重要：

- 最重要：最大字号 + 最深颜色 + 最大面积
- 次要内容：中等字号 + 中灰颜色
- 辅助说明：最小字号 + 浅灰颜色

避免"一屏同等大小的文字"——没有层级就没有重点。

### 色彩作为语义载体

颜色不只是装饰，要让颜色承载含义：
- **强调色**（如 `#4A9FD4`）：最重要的元素、推荐选项、当前步骤
- **中性色**（白/浅灰）：背景、次要卡片、分隔线
- **深色**（`#1E3A5F`）：标题栏、文字主色、边框
- **语义色**（可选）：绿色=正向/增长，红色=风险/下降，橙色=警示

每张幻灯片控制在 **3 种颜色以内**（不含黑/白/灰），整套演示稿的颜色体系保持一致。

### 呼吸感与空白

**不要把每寸空间都填满。** 适当的空白让眼睛休息，也让重要内容更突出：
- 每个元素之间留至少 16px 间距
- 文字与其容器边缘（卡片内边距）至少 16px
- 整体内容区距 viewBox 边缘至少 40px

### 对齐与网格

所有同类元素必须对齐——对齐是专业感的最重要来源：
- 多列卡片的 y/height 完全一致
- 多行文字的 x 坐标一致
- 居中元素用 `text-anchor="middle"` + 精确计算的 x 坐标

### 用形状强化结构含义

不要只用矩形和文字。形状本身可以传达意义：
- **圆形**：周期、整体、完整性
- **箭头**：方向、因果、转化
- **菱形**：决策点、核心概念
- **三角形**：层级、稳定性、方向
- **连线**：关联、流动、依赖

用基本 SVG 形状（`<polygon>` `<path>` `<circle>`）构建有意义的图标和示意图，而非空白色块。

## SVG 构建实用技巧

### 文字必须手动换行

SVG 中文字不会自动折行。超过约 20 个汉字（或 36 个英文字符）的文本必须拆分：

```xml
<!-- 错误：单行过长 -->
<text x="100" y="100" font-size="16" fill="#333">这是一段很长很长的文字，超出容器宽度后会直接溢出到边界之外</text>

<!-- 正确：手动换行 -->
<text x="100" y="100" font-size="16" fill="#333">这是一段很长的文字，</text>
<text x="100" y="122" font-size="16" fill="#333">超出后手动在下一行继续书写。</text>
<!-- 行间距 = font-size × 1.4，16px 字号行距约 22px -->
```

### 估算文字宽度（防越界）

生成坐标前先心算，确保文字不超出 viewBox：

- 中文：每字宽度 ≈ `font-size × 1.0` px
- 英文大写：每字母 ≈ `font-size × 0.7` px；小写 ≈ `font-size × 0.55` px
- `text-anchor="middle"` 时：中心 x ± 总宽度/2，两端都需 ≥ 40 且 ≤ 920

示例：`font-size="20"` 的 10 个汉字 ≈ 200px 宽。若 `text-anchor="middle" x="480"`，左端 = 380，右端 = 580，均在安全区内。

### 用 linearGradient 增加质感

渐变可以让色块更有层次感（限 ≤3 色标）：

```xml
<defs>
  <linearGradient id="headerGrad" x1="0" y1="0" x2="1" y2="0">
    <stop offset="0%" stop-color="#1E3A5F"/>
    <stop offset="100%" stop-color="#2D6A9F"/>
  </linearGradient>
</defs>
<rect width="960" height="64" fill="url(#headerGrad)"/>
```

### 构建图标：用基本形状组合

不要嵌入 `<image>`，用 `<g>` 组合基本形状：

```xml
<!-- 图表图标（3根柱子） -->
<g transform="translate(100, 200)">
  <rect x="0"  y="30" width="16" height="30" fill="#4A9FD4"/>
  <rect x="20" y="10" width="16" height="50" fill="#1E3A5F"/>
  <rect x="40" y="20" width="16" height="40" fill="#4A9FD4"/>
  <line x1="0" y1="60" x2="56" y2="60" stroke="#CBD5E0" stroke-width="1"/>
</g>
```

### 弧形箭头（循环结构用）

```xml
<!-- 顺时针弧形箭头，从角度0到角度270，圆心(480,300)，半径120 -->
<path d="M 600,300 A 120,120 0 0,1 480,420" stroke="#4A9FD4" stroke-width="3"
      fill="none" marker-end="url(#arrow)"/>
<defs>
  <marker id="arrow" markerWidth="8" markerHeight="8" refX="6" refY="3" orient="auto">
    <path d="M0,0 L0,6 L8,3 z" fill="#4A9FD4"/>
  </marker>
</defs>
```

## 常见错误与修复

| 问题 | 根本原因 | 修复方式 |
|------|----------|----------|
| 内容看起来很小，四周大片空白 | SVG 内容没铺满 viewBox | 先加铺满背景 `<rect width="960" height="540">`，再把元素坐标扩展到 40–920 范围 |
| 文字被截断 / 溢出右边界 | 单行文字过长，未手动换行 | 按"每字=font-size"估算宽度，超过容器宽度则拆行 |
| 形状超出幻灯片边缘 | 坐标计算错误，右/下坐标超过 960/540 | 检查：rect 的 x+width ≤ 960，circle 的 cx+r ≤ 960 |
| 整个 SVG 变成一张不可编辑的图片 | 使用了 `<filter>` / `<mask>` / Web 字体 | 删除这些元素；字体改为微软雅黑 / Arial |
| 颜色或样式不生效 | 样式写在 `<style>` 块内 | 改为内联 `style=` 属性或元素属性 |
| 渐变消失 | 使用了 `radialGradient` 或超过 3 色标 | 改用 `linearGradient` + ≤3 色标 |
| 各幻灯片风格不统一 | 每张独立选色，无统一色板 | 在第一张确定主色/强调色，其余幻灯片复用相同颜色变量 |
| 视觉结构和内容不匹配 | 套用了不合适的模板 | 回到"内容分析"步骤，重新判断内容关系类型 |

## 用户布局库（User Layout Templates）

以下 5 种布局来自用户实际使用的 PPT 模板，是经过业务验证的版式。**生成幻灯片时优先从这里匹配场景，再考虑上方的通用结构**。

所有坐标均为 960×540 坐标系。生成时严格遵守技术约束：无 `<filter>`、无 `radialGradient`、无 `clip-path`、无 `<use>` 跨引用、无根元素 `style=` 字体声明（字体写在每个 `<text>` 元素的 `font-family` 属性上）。

### UL-1：三列对比 + 中心双圆

**适用场景**：两大类别深度对比、双核心概念并排展开（功能A vs 功能B、左侧特性 vs 右侧架构）。

**结构概览**：
```
[主标题]
[左列：深蓝标题 + 要点列表]  [中心：两个相交圆]  [右列：浅蓝标题 + 要点列表]
```

**960×540 关键坐标**：
- 背景：`<rect width="960" height="540" fill="#F4F7FB"/>`（或浅灰 linearGradient）
- 顶部大标题：x=60, y=60，`font-size="26"` `font-weight="900"` `fill="#0A4B95"`
- 左列内容区：x=60–300，y=100–520；子标题 `font-size="15"` `fill="#1B5A9E"` `font-weight="bold"`，条目 `font-size="12"` `fill="#333333"`，圆点 `<circle r="3" fill="#1B5A9E"/>`
- 右列内容区：x=640–920，y=100–520；子标题 `fill="#30B5DF"`
- 中心双圆：圆A `cx=390 cy=310 r=82`，圆B `cx=570 cy=310 r=82`（两圆叠加，用 linearGradient 蓝色填充，`opacity="0.95"`）
- 圆内文字：`text-anchor="middle"` `font-size="21"` `fill="#FFFFFF"` `font-weight="bold"`，两行（dy="0" / dy="28"）

**设计要点**：双圆填充用 linearGradient（x1="0%" y1="0%" x2="100%" y2="100%"），不用 radialGradient；背景可加斜切装饰 path（浅灰，`opacity="0.5"`）增加层次感。

### UL-2：5 卡片问题矩阵（3 上 + 2 下）

**适用场景**：5 大问题/挑战/风险并列展示；分类陈述现状痛点；关键发现汇总。

**结构概览**：
```
[卡1 蓝] [卡2 红] [卡3 青]   ← 3 张等宽卡，各含 3 条问题
     [卡4 橙]    [卡5 绿]    ← 2 张较宽卡，各含 3 条问题
```

**960×540 关键坐标**：
- 顶部标题：x=60, y=52，`font-size="27"` `font-weight="900"`
- 顶部3卡：`width=263 height=210`，`rx=9`；x=40（卡1）, 325（卡2）, 610（卡3）；y=75
  - **卡片顶色条实现（禁用 clip-path）**：先画全圆角卡 `<rect rx=9>`，再叠画一个上部矩形 `<rect height=42 fill="色" rx=0/>` + 一个小圆角盖住底部棱角 `<rect y=21 height=21 fill="色"/>` ——或最简单：把色条 `height=42`，并把它的 y 与卡片同起点，宽高完全覆盖圆角上半部
  - 更简方案：色条就是一个无圆角的 `<rect rx=0>`，视觉上顶部两角方形、底部过渡到卡片内容，完全合规
- 卡内标题：`text-anchor="middle"` x=各卡中心，`font-size="13"` `fill="#FFFFFF"` `font-weight="bold"`；序号大字（"01"~"03"）`font-size="24"` `fill="#FFFFFF"` `opacity="0.2"` 放色条右侧
- 卡内条目：主标题行 `font-size="10.5"` `font-weight="bold"` `fill="（各卡强调深色）"`；细节行 `font-size="10"` `fill="#475569"`；每条最多 2 行，行距 15
- 底部2卡：`width=408 height=157`，`rx=9`；x=72（卡4）, 490（卡5）；y=300
- 各卡顶色条颜色：蓝 `#1E40AF`、红 `#DC2626`、青 `#0891B2`、橙 `#EA580C`、绿 `#059669`
- 卡片边框：`stroke="#E2E8F0"` `stroke-width="1"` 代替阴影（禁用 filter）

### UL-3：中心辐射架构图（轨道 + 卫星 + 左右标注）

**适用场景**：系统架构、核心平台 + 外围能力、防御层次体系、多维度解决方案辐射。

**结构概览**：
```
[标注A] ←─── ○  ───→ [标注C]
[标注B] ←─ ◉核心 ─→ [标注D]
              ○        [标注E]
```

**960×540 关键坐标**：
- 顶部标题：x=60, y=52
- 摘要条（可选）：x=40, y=68, width=880, height=52，`rx=9`，浅蓝背景 `#EBF4FF`；内含两行说明文字 `font-size="12"` `fill="#334155"`
- 中心组（用 `<g transform="translate(480, 330)">`）：
  - 轨道圆：`r=120`，`stroke="#93C5FD"` `stroke-dasharray="5,5"` `fill="none"` `stroke-width="1.5"`
  - 轨道方向小箭头：4 个 `<polygon>` 小三角，分布在 45°/135°/225°/315° 位置，`fill="#60A5FA"`
  - 中心蓝圆：`r=64`，linearGradient（`#60A5FA` → `#1D4ED8`）
  - 中心文字：两行，`font-size="18"` `font-weight="900"` `fill="#FFFFFF"`
  - 卫星圆（5个，r=18）：均匀分布在轨道上，`fill="#3B82F6"` `stroke="#FFFFFF"` `stroke-width="3"`（stroke 代替阴影）
- 左侧标注（text-anchor="end"，均为绝对坐标）：
  - 标注1：x≈345, y=245（中心左上）；标注2：x≈360, y=405（中心左下）
- 右侧标注（text-anchor="start"）：
  - 标注3：x≈580, y=230；标注4：x≈625, y=330；标注5：x≈572, y=428
- 每组标注：主标题 `font-size="15"` `fill="#1E3A8A"` `font-weight="bold"`，副文字2行 `font-size="10.5"` `fill="#475569"`，行距14

**设计要点**：无 filter、无 radialGradient；中心圆 linearGradient；卫星圆用 stroke 显示"浮出"感。

### UL-4：层级价值架构（顶部目标 Banner + 双层 2×2 卡片）

**适用场景**：价值主张（效率层 + 安全层）、战略目标分解（顶部总目标 + 两层四大支柱）、分层架构说明。

**结构概览**：
```
╔═══════════════[ULTIMATE GOAL]═══════════════╗
║ [效率/驱动层标签]                            ║
║   [卡A]  ↑   [卡B]                          ║
║ [基础/安全层标签]                            ║
║   [卡C]  ↑   [卡D]                          ║
╚══════════════════════════════════════════════╝
```

**960×540 关键坐标**：
- 顶部标题：x=60, y=46，`font-size="27"` `font-weight="900"` `fill="#1E3A8A"`
